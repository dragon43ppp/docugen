# Code of Conduct

## Our Goal

We want this project to be a welcoming place for people building practical document workflows.

## Expected Behavior

- Be respectful and constructive
- Focus on the problem, not the person
- Share reproducible steps when reporting issues
- Protect private data, credentials, and customer documents

## Unacceptable Behavior

- Harassment or personal attacks
- Publishing private API keys, internal endpoints, or confidential documents
- Spam, abuse, or deliberately disruptive behavior

## Enforcement

Project maintainers may moderate discussions, issues, and pull requests to keep collaboration productive and safe.
