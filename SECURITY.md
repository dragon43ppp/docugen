# Security Policy

## Supported Usage

This repository is intended for local or self-hosted use with your own API configuration.

## Reporting a Vulnerability

If you discover a security issue, please do not publish secrets, tokens, or exploit details in a public GitHub issue.

Please include:

- affected file or feature
- reproduction steps
- potential impact
- whether the issue involves frontend config storage, backend request proxying, or file processing

## Sensitive Data Rules

- Never commit real API keys
- Never commit private upstream endpoint credentials
- Remove customer files and sensitive documents from screenshots and bug reports
- Redact logs before sharing them
